# 手機響應式設計說明

## 概述

本專案已完整支援手機、平板和桌面設備的響應式顯示，使用 Tailwind CSS 的響應式斷點來自動適配不同螢幕尺寸。

## 支援的設備尺寸

### 📱 手機 (寬度 < 640px)

- **圖表高度**: 300px
- **標題大小**: text-xl (較小)
- **內距**: 減少 padding (p-3)
- **按鈕**: 全寬顯示，較小的文字和圖示
- **網格佈局**: 單列顯示所有卡片

### 📱 平板 (640px ≤ 寬度 < 1024px)

- **圖表高度**: 400px
- **標題大小**: text-2xl (中等)
- **內距**: 標準 padding (p-4)
- **網格佈局**: 2列顯示統計卡片

### 💻 桌面 (寬度 ≥ 1024px)

- **圖表高度**: 500px
- **標題大小**: text-3xl (完整)
- **內距**: 完整 padding (p-6)
- **網格佈局**: 3列顯示日期選擇器，4列顯示統計卡片

## 響應式特性

### 1. Header (頂部標題區)

- 手機: 垂直堆疊標題和按鈕
- 桌面: 水平排列標題和按鈕

### 2. 日期範圍選擇器

- 手機/平板: 單列顯示日期輸入框和套用按鈕
- 桌面: 3列網格排列

### 3. K線圖表

- 動態調整高度適應不同螢幕
- 保持完整功能（縮放、平移、點擊警報等）
- 圖表標籤和按鈕根據螢幕大小調整

### 4. 下跌警報

- 響應式卡片大小和間距
- 警報按鈕根據螢幕調整大小
- 保持點擊功能正常運作

### 5. 統計資訊

- 手機: 單列顯示
- 平板: 2列顯示
- 桌面: 4列顯示

## Tailwind CSS 響應式類別

使用的主要響應式類別：

- `sm:` - 適用於寬度 ≥ 640px
- `md:` - 適用於寬度 ≥ 768px
- `lg:` - 適用於寬度 ≥ 1024px

### 範例

```vue
<!-- 手機上 text-xl，桌面上 text-3xl -->
<h1 class="text-xl sm:text-2xl lg:text-3xl">

<!-- 手機上 p-3，平板上 p-4，桌面上 p-6 -->
<div class="p-3 sm:p-4 lg:p-6">

<!-- 手機上單列，平板上 2 列，桌面上 4 列 -->
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4">
```

## 最佳實踐

1. **觸控友好**: 按鈕和可點擊元素有足夠的大小 (至少 44x44px)
2. **可讀性**: 文字大小在小螢幕上仍保持可讀
3. **滾動優化**: 長列表可以流暢滾動
4. **性能**: 圖表在不同尺寸下都能保持良好性能

## 開發指南

修改響應式樣式時：

1. 優先使用 Tailwind 的響應式類別
2. 在 ChartView.vue 中的 `chartStyle` 計算屬性調整圖表高度
3. 使用 Chrome DevTools 的設備模擬器測試
4. 確保所有互動功能在觸控設備上正常工作
